During game project 7 we had a need to view and edit components from our editor, and since we ideally wanted to keep our editor/game projects separated which meant no component-specific code, I was tasked with making a reflection system.

Before starting I made a list of requirements:

  • No extra data stored on components, copying and moving components had to remain fast.
  • No type specific editing code needed, all types had to recursivly render
  • Support for common stl-container like vector and unordered_map

Registration

With the first requirement in mind, I decided on creating a registry where you register a type's members once, which you can later query using their registred name or an instance of the object.
